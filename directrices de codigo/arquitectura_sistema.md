Arquitectura del Sistema:
El Sistema de Almacén UNEA fue diseñado con una arquitectura híbrida que combina lo mejor del desarrollo web y de escritorio. Se utilizó Tauri como framework principal, que permite crear aplicaciones nativas multiplataforma utilizando tecnologías web para el frontend y Rust para el backend. Esta elección fue estratégica porque ofrece un rendimiento superior comparado con soluciones como Electron, consumiendo menos recursos del sistema y proporcionando acceso directo al hardware.

El frontend se desarrolló con React, aprovechando su ecosistema maduro para crear interfaces de usuario dinámicas y responsivas. Se implementó un sistema de componentes reutilizables que facilita el mantenimiento y la escalabilidad. La gestión de estado se maneja mediante Context API, permitiendo compartir datos entre componentes sin prop drilling excesivo. Para el enrutamiento, se utilizó React Router, que mantiene el estado de navegación y proporciona transiciones fluidas entre vistas.

En el backend, Rust proporciona una base sólida y segura para las operaciones críticas. Se implementaron módulos especializados para cada funcionalidad: manejo de bases de datos con Rusqlite, operaciones de sistema de archivos con dirs, y comunicación IPC a través de Tauri's invoke system. La serialización de datos se realiza con Serde, asegurando una conversión eficiente entre tipos de datos Rust y JavaScript. Esta arquitectura separada garantiza que las operaciones sensibles se ejecuten de manera segura y eficiente.

La integración entre frontend y backend se logra mediante comandos Tauri, que permiten llamadas seguras desde JavaScript a funciones Rust. Esto crea una comunicación bidireccional robusta sin comprometer la seguridad. El sistema de archivos se maneja con permisos específicos configurados en Tauri, permitiendo operaciones de importación/exportación controladas. En conjunto, esta arquitectura proporciona una aplicación robusta, segura y eficiente que cumple con los estándares modernos de desarrollo de software.
