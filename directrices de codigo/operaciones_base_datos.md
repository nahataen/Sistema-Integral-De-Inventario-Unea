Operaciones de Base de Datos:
El corazón del Sistema de Almacén UNEA reside en sus capacidades avanzadas de gestión de bases de datos SQLite. Se desarrolló un sistema completo que permite a los usuarios manejar múltiples bases de datos de manera intuitiva y segura. La funcionalidad principal incluye la importación y exportación de bases de datos completas, con validaciones automáticas que verifican la integridad de los datos antes de cargarlos al sistema.

Se implementaron operaciones CRUD completas para tablas y registros, permitiendo crear, leer, actualizar y eliminar datos de forma controlada. Cada operación incluye confirmaciones y validaciones para prevenir errores accidentales. Por ejemplo, al eliminar una base de datos o tabla, el sistema requiere confirmación explícita del usuario, reduciendo el riesgo de pérdida de datos. Las operaciones de edición se realizan en contextos seguros, donde solo el registro seleccionado se modifica, manteniendo la integridad del resto de los datos.

Una característica destacada es el manejo de imágenes en los registros. El sistema permite subir archivos de imagen, convertirlos a formato Base64 para almacenamiento en la base de datos, y generar miniaturas automáticas para una visualización eficiente. Esto es particularmente útil para inventarios donde las imágenes facilitan la identificación rápida de productos. Las validaciones aseguran que solo se acepten formatos de imagen válidos y tamaños razonables.

La gestión de esquemas de tabla es dinámica, permitiendo agregar o eliminar columnas en tiempo real. Esto proporciona flexibilidad para adaptar las estructuras de datos según las necesidades cambiantes del inventario. Una característica importante es la validación de nombres únicos para tablas: si se intenta crear una nueva tabla con un nombre que ya existe en la base de datos, el sistema informa al usuario que debe utilizar otro nombre, previniendo conflictos y manteniendo la integridad de los datos. Todas las operaciones incluyen logging y manejo de errores robusto, asegurando que cualquier problema se reporte claramente al usuario sin comprometer la estabilidad del sistema. En conjunto, estas funcionalidades hacen que la gestión de bases de datos sea eficiente, segura y adaptable a diversos escenarios de uso.
